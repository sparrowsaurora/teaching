# Comments

youve seen comments in python already,

comments are any sentences prefixed with a '#'

if you're comming from another language you might be used to having two forms of comments

the line comment, which in python is the hashtag

and the multi-line or docblock comment, which python does not have  
to remedy this, the common convention is to use a column of line comments for header docs,  
and the use **string literal blocks** or string docblocks.  
we use these for functions:

```python
def func_name():
    """
    text
    text
    """
    # this is also a comment
```

and you write them using 3 single or double quotes in a row.  
if you are using VSCode, like i am, this will be interperated as a comment by the program. however it is technically not a comment.  
this is the main convetion for documentation in python
